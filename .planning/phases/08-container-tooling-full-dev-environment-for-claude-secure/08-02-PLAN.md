Create `tests/run-all.sh` with the following implementation:

```bash
#!/bin/bash
# run-all.sh -- Unified test runner for claude-secure
# Runs all test-phase*.sh files in sequence, manages container lifecycle,
# and reports a summary of results.
#
# Usage:
#   bash tests/run-all.sh           # Run all tests, tear down after
#   bash tests/run-all.sh --keep    # Run all tests, keep containers running
#
# Exit 0 if all phases pass, exit 1 if any fail.
set -uo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_DIR="$(dirname "$SCRIPT_DIR")"

KEEP_RUNNING=false
if [[ "${1:-}" == "--keep" ]]; then
  KEEP_RUNNING=true
fi

cd "$PROJECT_DIR" || exit 1

FAILED_PHASES=()
PASSED_PHASES=()
TOTAL=0

cleanup() {
  if [ "$KEEP_RUNNING" = false ]; then
    echo ""
    echo "Tearing down containers..."
    docker compose down >/dev/null 2>&1 || true
  else
    echo ""
    echo "Containers left running (--keep flag)."
  fi
}
trap cleanup EXIT

echo "========================================"
echo "  claude-secure -- Full Test Suite"
echo "========================================"
echo ""

for test_file in "$SCRIPT_DIR"/test-phase*.sh; do
  if [ ! -f "$test_file" ]; then
    echo "No test files found."
    exit 1
  fi

  phase_name=$(basename "$test_file" .sh)
  ((TOTAL++))

  echo ""
  echo "========================================"
  echo "  Running: $phase_name"
  echo "========================================"

  if bash "$test_file"; then
    PASSED_PHASES+=("$phase_name")
  else
    FAILED_PHASES+=("$phase_name")
  fi
done

echo ""
echo "========================================"
echo "  Summary"
echo "========================================"
echo ""
echo "  Total:  $TOTAL phases"
echo "  Passed: ${#PASSED_PHASES[@]}"
echo "  Failed: ${#FAILED_PHASES[@]}"

if [ ${#FAILED_PHASES[@]} -gt 0 ]; then
  echo ""
  echo "  Failed phases:"
  for p in "${FAILED_PHASES[@]}"; do
    echo "    - $p"
  done
  echo ""
  exit 1
fi

echo ""
echo "  All phases passed."
exit 0
```

Key design decisions:
- Each test-phase*.sh manages its OWN container lifecycle (they already do `docker compose up -d` and `docker compose down` internally). The unified runner does NOT pre-start containers because each phase test has different env var requirements (e.g., test-phase7.sh sets SECRETS_FILE to a temp file, test-phase2.sh needs different setup). The runner only does teardown cleanup.
- `--keep` flag skips teardown for local dev convenience (containers may already be down from last test's cleanup, but this prevents a redundant final `docker compose down`).
- Tests run in glob order (test-phase1.sh, test-phase2.sh, ..., test-phase7.sh) which respects dependency order.
- The runner collects phase names into PASSED/FAILED arrays for the summary.
- Exit code 1 if any phase fails, 0 if all pass.
